    Work

    This section contains short summaries of all the Portfolio Documents as well as links to the documents themselves, when possible.     Project 1 - Mozilla Article

    Contributed: August 2020

    Project Summary: Rewrote the webpage printing for the Mac segment of the page to fix inaccuracies.

    Process: Using the previous Mac Printing article as a template, I removed the old information and added new information by opening Firefox on my personal Mac computer and moving through the printing process. Once the draft work was complete, I used editing programs to check the work for errors, then sent it in for review.

    Result: Mac Printing changes were accepted; however, the changes made may no longer be available because of further changes to the program.

    Project Link: MozillaPrinterArticleImages

    Project 2 - PB&J SOP

    Contributed: October 2022

    Project Summary: This was practice creating an SOP using a commonly created lunch item as the SOPs subject.

    Process: Firstly, as this was the first construction of this kind of document, a day was used to research the specifics of what is required for a proper SOP. This was a combination of looking up written and video guides and checking government sites that require their SOPs to be published. After examining these resources and sites, a general template was put together. Following that, the description and overall writing style were chosen. With all of this set, the SOP itself was finally constructed. Once the draft was complete, the writing was reviewed and put through editing programs to confirm the grammatical accuracy of the document's contents.

    Result: The SOP is a detailed description of how to assemble a PB&J sandwich, including specific techniques for creation that may benefit an individual with little experience in sandwich making.

    Project Link: PeanutButterSOP

    Project 3 - State Profiles

    Contributed: March 2022

    Project Summary: FEMA Region 7 State Profiles to aid in Disaster Response and Recovery.

    Process: This previously unfinished project needed to be updated, completed, and standardized. It started with the collection of old resources used by the project, including the templates put together by the previous team. After putting these together, investigating the resources, and consulting the client, we picked one template as a base and began filing in information. Because this project relied on information not necessarily discoverable in public domain, we coordinated with other individuals within the organization to obtain information to fill in the documentation. Our contract ended with us having filled in the majority of the information and completing some edits suggested by the contractor to improve the work. Weekly meetings with said contractor were held to provide updates, put in RFIs, and clarify questions.

    Result: My and my team's work on this project received verbal praise multiple times from the contractor and other members of the contractor's team. The document serves as an aid to any new individuals coming into the region so that they can quickly familiarize themselves with the necessary information.

    As this project is FOUO, I am unable to link the project work at this time.

    Project 4 - Control Panel

    Contributed: July 2022

    Project Summary: This is a fictional Control Script guide for piloting droid ships from the Star Wars universe.

    Process: This was not created by researching other control scripts but by analyzing REST API documentation. After looking through multiple REST API documents, the concept of the program itself, as well as how to document it, was created. After putting together a general template based loosely on the API documentation and deciding on a style guide, the project was put together. As this was documentation for a fictional project, there were equal amounts of editing the project itself as there was editing the documentation.

    Result: The result is a guide to a fictional tool, allowing individuals to use the guide to make the "tool" as a game similar to Zork, should they wish to.

    Project Link: CISDroidAttackStarfighterControlProgram

    Project 5 - MadCap Flare Practice

    Contributed: November 2022

    Project Summary: Disney + user guide meant to make using and navigating the Disney + webpage easier.

    Process: After researching various user guides, a PDF template was selected from those available in the MadCap Flare documentation tool. After filling out the template to match a user guide, an already present Disney + account and various Disney + videos were used to fill in the crucial details for each category. After the initial draft was completed, editing programs were used to aid in the editing process of the documentation. Afterward, as the trial version of MadCap Flare scrambles finished documents, the pdf was inserted into a PDF editor. After completion, the PDF was put through another image editing program to finalize changes to the document.

    Result: A mostly polished Disney + user guide. The mouse got caught in the screenshot on some of the PDF pictures used in the project. Besides that, it will aid new users of Disney + in navigating some of the trickier parts of the webpage.

    Project Link: UnofficialDisneyPlusBGuidePartial

    About

    I'm Luke G. Hendley, a communications professional specializing in Interpersonal and Written Communications. What all of that translates to is that I'm a professional communicator with a knack for small group communications and for communicating complex concepts in a way someone will understand using writing.

    I graduated from North Carolina State University with a BA in Interpersonal Communications. After some volunteer technical writing and editing, I worked as a FEMA Corps Member. I was essentially an intern for the Federal Emergency Management Agency.

    While working in FEMA Corps, I spent near half a year working on State and Tribal profiles, documents recording everything an emergency management worker would need to know about a state or tribe so that they can quickly start a disaster response mission and help them.

    Since graduating from the FEMA Corps program, I've been putting together technical writing projects to help further demonstrate my ability as a technical writer. These projects are much of what makes up the works presented on this site.
